# Plan 01-03: Navigation Shell and Responsive Layout

## Goal
Create a responsive navigation shell with links to dashboard, habits, and analytics pages. Ensure mobile-first responsive design.

## Steps

1. Create `src/components/layout/navbar.tsx` - navigation bar component:
   - Logo/app name "HabitTracker"
   - Desktop: horizontal nav links (Dashboard, Habits, Analytics)
   - Mobile: hamburger menu with Sheet (slide-out) from shadcn/ui
   - Active link highlighting using `usePathname()`
2. Create `src/components/providers.tsx` - theme provider wrapper (placeholder for next-themes, set up in UX phase)
3. Update `src/app/layout.tsx` - integrate navbar and providers into root layout
4. Create stub pages:
   - `src/app/page.tsx` - Dashboard page (placeholder with title)
   - `src/app/habits/page.tsx` - Habits management page (placeholder)
   - `src/app/analytics/page.tsx` - Analytics page (placeholder)
5. Add responsive styles:
   - Mobile: bottom navigation or hamburger menu
   - Desktop: sidebar or top navigation bar
   - Proper spacing and padding at all breakpoints

## Files Created/Modified
- `src/components/layout/navbar.tsx` - navigation component
- `src/components/providers.tsx` - provider wrapper
- `src/app/layout.tsx` - root layout with navigation
- `src/app/page.tsx` - dashboard stub
- `src/app/habits/page.tsx` - habits stub
- `src/app/analytics/page.tsx` - analytics stub

## Verification
- Navigation renders with 3 links (Dashboard, Habits, Analytics)
- Clicking links navigates to correct pages
- Mobile view shows hamburger/sheet navigation
- Desktop view shows horizontal navigation
- All pages render without errors

## Decision
- Use top navigation bar for simplicity (sidebar adds complexity for v1)
- Use shadcn/ui Sheet component for mobile menu
- Active link highlighted with different color/style
