# Plan 01-01: Next.js Project Setup

## Goal
Bootstrap a Next.js 15 project with TypeScript, Tailwind CSS, and shadcn/ui fully configured and running locally.

## Steps

1. Create Next.js 15 project using `create-next-app` with TypeScript, Tailwind CSS, ESLint, App Router, and src directory
2. Install supporting dependencies: date-fns, lucide-react, clsx, tailwind-merge, zod, next-themes
3. Initialize shadcn/ui with `npx shadcn@latest init`
4. Add core shadcn/ui components: button, card, checkbox, input, dialog, dropdown-menu, badge, separator, sheet, navigation-menu
5. Create utility file `src/lib/utils.ts` with `cn()` helper (if not auto-created by shadcn)
6. Verify the app starts with `npm run dev` and renders the default page

## Files Created/Modified
- `package.json` - project dependencies
- `tsconfig.json` - TypeScript config
- `tailwind.config.ts` - Tailwind config
- `src/app/layout.tsx` - root layout
- `src/app/page.tsx` - home page
- `src/app/globals.css` - global styles with Tailwind imports
- `src/lib/utils.ts` - utility functions
- `src/components/ui/` - shadcn/ui components

## Verification
- `npm run dev` starts without errors
- Visiting http://localhost:3000 shows a page
- TypeScript compiles without errors (`npx tsc --noEmit`)

## Decision
- Use local PostgreSQL instead of Vercel Postgres for local development
- Use `pg` package with `drizzle-orm/node-postgres` adapter for local dev
- Environment variables will be configured to swap between local and Vercel Postgres
